# 프로젝트 기획서: 로보어드바이저 신뢰성 검증 테스트베드

## 목적
투자 알고리즘(로보어드바이저) 자체를 만드는 게 아니라, 어떤 투자 알고리즘이든 끼워 넣으면 신뢰성을 자동으로 검증해주는 평가 인프라를 구현한다. 코스콤이 운영하는 RA테스트베드 개념을 이해하고 직접 구현해보는 것이 목표이며, 백엔드 시스템 설계 역량과 더불어 데이터 분석/금융 도메인 이해를 보여주는 포트폴리오로 활용한다.

## 전체 흐름
투자 전략(plug-in) → 과거 시장 데이터(국면 분류) → 백테스트 엔진 → 리스크 지표 계산 → 신뢰성 리포트(Streamlit 대시보드)

## 기술 스택
- Python 3.12, pandas, numpy, yfinance, pytest
- Streamlit (Phase1~2 UI), FastAPI는 Phase3 스트레치
- Plotly/matplotlib (시각화)
- 배포: Streamlit Community Cloud
- DB 미사용: 과거 시세는 parquet/csv 캐싱, 결과는 즉시 계산 — 의도적 단순화이며 이유를 README에 명시

## 핵심 설계 결정 (먼저 합의할 것)

### 1. 전략 인터페이스
모든 전략은 동일한 형태로 들어와야 한다: 특정 시점까지의 시장 데이터를 받아 자산별 목표 비중(dict)을 반환하는 함수/클래스. 이 표준 포맷 덕분에 규칙 기반 전략이든 ML 기반 전략이든 동일한 백테스트 엔진에서 평가할 수 있다.

### 2. lookahead bias 방지 (가장 중요한 설계 결정)
전략이 "오늘" 시점에 내린 결정은 반드시 그날 종가까지의 정보만 사용해야 하고, 실제 거래 체결은 다음 거래일 시가에 일어나는 것으로 분리한다. 신호 계산 시점과 체결 시점을 명확히 나눠서, 전략이 미래 데이터를 참조할 수 없는 구조를 코드 레벨에서 강제한다.

### 3. 거래비용/슬리피지 모델링
매 리밸런싱마다 거래 비용(예: 0.1% 가정)을 반영해 실제와 가까운 수익률을 계산한다. Phase1은 고정 비율 비용으로 단순화하고, 이 단순화의 한계를 README에 명시한다.

### 4. 시장 국면 분류 기준
객관적 규칙(예: 정점 대비 -20% 이상 하락 구간을 하락장으로 정의)으로 국면을 나누고, 2008 금융위기·2020 코로나 급락·2022 금리인상기 등 대표 구간을 사전 정의해둔다.

### 5. 통과/경고 판정 기준
리스크 지표별로 임계값(예: MDD -40% 초과 시 경고)을 설정하고, 그 기준을 왜 그렇게 정했는지 근거를 README에 남긴다.

## 단계별 범위

### Phase 1 — MVP (먼저 끝낼 것)
- 전략 인터페이스 정의 + 예시 전략 2~3개 (예: 고정비중 60/40, 이동평균 교차)
- 과거 시장 데이터 수집(yfinance) + 로컬 캐싱
- 백테스트 엔진: 일별 시뮬레이션, lookahead bias 방지 구조, 거래비용 반영
- 기본 리스크 지표: CAGR, MDD, 샤프비율, 변동성
- Streamlit 기본 UI: 전략 선택 → 실행 → 수익곡선/지표 표시

### Phase 2 — 차별화 기능 (Phase1 완료 후)
- 시장 국면 분류 + 국면별 성과 분해
- 시나리오 버튼 UI (2008 / 2020 / 2022 클릭 시 해당 구간만 즉시 평가)
- 표준화된 신뢰성 리포트(통과/경고 항목)
- 사용자 전략 업로드 기능(간단한 포맷)

### Phase 3 — 스트레치 (시간 남으면)
- 여러 전략 동시 비교 시각화
- VaR/CVaR 등 고급 리스크 지표
- lookahead bias 자동 탐지 가드레일 (전략이 미래 데이터를 참조하는지 코드로 검증)
- FastAPI로 백테스트 엔진을 REST API로 노출

## 개발 순서 권장
1. 전략 인터페이스 + 예시 전략을 단위 테스트와 함께 구현 (TDD)
2. 데이터 로더(수집 + 캐싱) 구현
3. 백테스트 엔진 구현 — lookahead bias 방지를 최우선으로 검증
4. 리스크 지표 계산 모듈
5. 국면 분류 + Phase2 기능
6. Streamlit UI 연결
7. 신뢰성 리포트 생성 → 배포
